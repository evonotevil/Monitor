【任务五：风险评估（仅 is_relevant=true 时执行）】
从以下四个维度评估该动态对中资出海游戏公司的风险等级，每项 0-3 整数：

risk_revenue（营收影响）：
  0 = 无直接营收影响
  1 = 间接影响（如需更新隐私政策，不影响收入流）
  2 = 可能罚款或部分业务受限（如某地区 IAP 受限、中等金额罚款）
  3 = 严重营收威胁（大额罚款、应用下架、支付通道被封、核心市场禁入）

risk_product（产品改动）：
  0 = 无需产品改动
  1 = 轻微改动（更新政策文案、调整隐私声明）
  2 = 中等改动（接入年龄验证、公示概率/保底机制、调整 IAP 流程、消费上限适配）
  3 = 重大改动（核心玩法重构、Gacha/Loot Box 机制大改、VIP 体系重构、下架特定功能模块）

risk_urgency（时间紧迫性）：
  0 = 无明确时间要求（早期讨论/信号）
  1 = 充裕时间（>6 个月后生效）
  2 = 中等紧迫（1-6 个月内需完成合规）
  3 = 紧急（已生效或 <1 个月，需立即行动）

risk_scope（影响范围）：
  0 = 单一非核心市场
  1 = 单一核心市场（如仅日本、仅韩国）
  2 = 多个核心市场或全区域性法规（如全欧盟、全东南亚）
  3 = 全球性影响或主要平台全局策略变更（如 App Store 全球政策）

⚠️ 评分原则：
- 基于文章实际内容评估，不要猜测或夸大
- 草案/早期讨论阶段 urgency 通常为 0-1
- 已生效/执法动态 urgency 通常为 2-3
- 单一国家的地方性政策 scope 通常为 0-1
