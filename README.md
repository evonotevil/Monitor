# 🌐 Lilith Legal · 全球游戏合规动态监控

> 面向合规团队的自动化情报工具，每天自动追踪全球主要市场的游戏监管动态，翻译整理后推送到飞书，并沉淀为可查阅的周报。

---

## 这个工具能帮你做什么

你不需要每天手动搜索各国监管机构的网站或新闻。这套系统会自动完成以下工作：

1. **每天早上 8 点**，自动从全球 50+ 个官方监管机构、法律情报平台和行业媒体抓取前一天的合规动态
2. **AI 自动翻译**，将英语、日语、韩语、越南语等多语言原文整理成规范的中文标题和摘要
3. **AI 风险评估**，从营收影响、产品改动、时间紧迫性、影响范围四个维度综合评分，帮助团队快速判断优先级
4. **过滤噪音**，自动排除硬件评测、电竞赛事、公司财报等无关内容
5. **工作日推送飞书日报**，按地区分类展示当日最重要的合规动态，高风险条目带有风险标签
6. **写入飞书多维表格**，供团队人工标注和筛选，含四维风险评分
7. **每周一自动发布周报**，生成可在手机和电脑上查看的 HTML 报告页面（也可手动触发）

## 飞书多维表格 — 团队工作台

多维表格是团队日常工作的核心入口。每天新抓取的条目会自动写入，初始状态为「🤖 待初筛」。

### 处理状态说明

| 状态 | 含义 | 动作 |
|------|------|------|
| 🤖 待初筛 | 系统自动写入，尚未人工确认 | 需要人工判断 |
| ✅ 推送 | 确认为有价值的合规动态 | 会出现在周报中 |
| 🗑️ 噪音/不推送 | 判断为无关或低价值 | 不出现在周报中 |
| 其他自定义状态 | 团队自行定义 | 会出现在周报中 |

> **重要**：只有状态不是「🤖 待初筛」和「🗑️ 噪音/不推送」的条目，才会被纳入周报。所以每周发布前，请确保团队已完成初筛。

### 主要字段说明

| 字段 | 说明 |
|------|------|
| 动态标题 | AI 生成的中文标题，格式为 [国家/地区] 事件描述 |
| 摘要 | AI 生成的 60-90 字合规摘要，说明具体规则和需要采取的行动 |
| 💡 核心结论 | 人工补充的法务判断（若填写，周报优先展示此字段而非摘要） |
| 国家/地区 | 按大区分组：北美 / 欧洲 / 日韩 / 港澳台 / 东南亚 / 中东 / 南美 / 大洋洲 / 其他 |
| 合规类别 | 见下方分类说明 |
| 处理状态 | 见上方状态说明 |
| 跟进 BP | 负责跟进的合规 BP |
| 发布日期 | 原始文章发布日期 |
| 原始链接 | 点击可查看原文 |
| 信源名称 | 来源媒体或机构名称 |
| 营收影响 | AI 评估的营收风险等级（0-3），0=无影响，3=下架/大额罚款 |
| 产品改动 | AI 评估的产品改动量（0-3），0=无需改动，3=核心玩法重构 |
| 时间紧迫性 | AI 评估的时间紧迫度（0-3），0=无时间要求，3=已生效/<1个月 |
| 影响范围 | AI 评估的影响范围（0-3），0=单一非核心市场，3=全球 |

> **风险评分字段**：需在多维表格中手动添加「营收影响」「产品改动」「时间紧迫性」「影响范围」四个数字列。未添加时系统正常运行，只是不写入这些字段。

## 周报在哪里看

每次发布周报后，可通过以下方式访问：

- **飞书卡片**：周报推送时会附带两个按钮，分别打开手机端和电脑端报告
- **手机端报告**：适合手机浏览，竖向排版
- **电脑端报告**：适合宽屏阅读，也是 PDF 打印的来源
- **历史周报**：所有历史周报均有存档，可通过报告页面底部的链接或直接访问存档目录查看
- **自动防缓存**：报告链接带有时间戳参数，HTML 页面也设置了禁止缓存的 meta 标签，点击飞书卡片按钮始终打开最新内容，无需手动清除浏览器缓存

## 日报卡片怎么看

飞书日报卡片会在每个工作日早上 8 点左右收到。无新增动态时也会推送一张绿色卡片告知「昨日无新增合规动态」。周一的日报会自动补上周六和周日的动态。

**卡片结构：**
- **顶部**：新增条数（周一显示「周末至今新增 N 条」），以及各地区动态数量概览
- **AI 综述**（灰色引用块）：150 字以内的整体情况总结
- **按地区分块**：每个有动态的地区单独一栏，展示最重要的 1-2 条；若该地区实际条数超过展示数，标题会显示「今日 N 条（展示 M 条）」
- **每条动态包含**：风险等级（🔴高 / 🟠中 / 🔵一般）、地区、合规类别、标题链接、机制变动摘要
- **风险标签**（仅 AI 评估的高风险条目）：如「营收🔴·紧迫⚠️」，表示该条目在营收影响和时间紧迫性维度 ≥ 2

**去重机制：** 同一事件被多家媒体报道时，日报只展示信源最权威的那条（官方机构 > 法律情报 > 行业媒体 > 普通新闻），避免重复

**卡片颜色含义：**
- 🟢 绿色 Header：无新增动态
- 🔴 红色 Header：当日有最高风险动态（影响评分 ≥ 9）
- 🟠 橙色 Header：当日有高风险动态（影响评分 ≥ 7）
- 🔵 蓝色 Header：一般重要性动态

## 风险评估机制

系统采用 **AI 四维评估 + 正则兜底** 的双轨评分机制，帮助团队快速判断每条动态的优先级。

### 四个风险维度（每项 0-3 分）

| 维度 | 0 | 1 | 2 | 3 |
|------|---|---|---|---|
| **营收影响** | 无直接影响 | 间接影响 | 罚款/部分受限 | 下架/大额罚款/支付被封 |
| **产品改动** | 无需改动 | 轻微（政策文案） | 中等（年龄验证/概率公示） | 重大（核心玩法重构） |
| **时间紧迫性** | 无时间要求 | >6 个月 | 1-6 个月 | 已生效或 <1 个月 |
| **影响范围** | 单一非核心市场 | 单一核心市场 | 多核心市场/全区域 | 全球/主要平台全局策略 |

### 综合评分

四个维度通过加权公式映射到 **1.0-10.0 分**（≥9 高风险 / ≥7 中风险 / ≥5 关注 / <5 低优先）。

AI 评估搭载在翻译+分类的同一次 LLM 调用中，不产生额外 API 费用。当 LLM 不可用时，自动退回基于关键词匹配的正则评分。

## 常见问题

**Q：今天没收到日报，是系统出问题了吗？**
A：工作日每天都会推送日报（无新增动态时也会推送绿色卡片）。如果完全没收到，可以去 GitHub Actions 页面查看当日运行日志确认。周末不推送飞书卡片，但数据照常抓取，周一日报会自动补上周末的动态。

**Q：多维表格里的条目看起来不相关，是 AI 判断错了吗？**
A：AI 有一定误判率。遇到这种情况，把处理状态改为「🗑️ 噪音/不推送」即可，这条内容就不会进入周报。如果某类噪音频繁出现，可以告知技术负责人添加过滤规则。系统对官方/法律信源已增加数字行业信号词过滤，会自动排除传统行业（食品、通信贩卖等）的监管新闻。

**Q：我在多维表格里填写了「💡 核心结论」，周报里会显示吗？**
A：会。如果填写了核心结论，周报里这条动态的摘要部分会优先显示核心结论的内容，而不是 AI 自动生成的摘要。

**Q：周报是自动发送的吗？**
A：否。周报需要团队在完成多维表格初筛后，手动在 GitHub Actions 页面点击「Run workflow」触发发布。

**Q：我想查某个关键词的历史动态，怎么做？**
A：可以直接在飞书多维表格里筛选，也可以联系技术负责人通过数据库查询。
